# Style Guide for Physical AI Book

This document outlines the formatting and style guidelines for the Physical AI Book content.

## Markdown Structure

### Document Frontmatter
Each document should include the following frontmatter:

```markdown
---
title: Title of the Document
description: Brief description of the content
sidebar_position: X
---
```

### Headers
- Use `#` for main headings (H1)
- Use `##` for section headings (H2)
- Use `###` for subsection headings (H3)
- Avoid using H4 and below unless absolutely necessary

### Code Blocks
- Use triple backticks (```) for code blocks
- Specify the language after the opening backticks for syntax highlighting
- Example: ```python, ```javascript, ```bash

### Images
- Store images in the `/static/img/` directory
- Use descriptive alt text for accessibility
- Add a caption below the image if needed

```markdown
![Description of image](/img/image-name.png)

*Caption for the image*
```

## Content Guidelines

### Learning Objectives
Each lesson should begin with a "Learning Objectives" section that clearly states what the reader will be able to do after completing the lesson.

### Prerequisites
When applicable, include a "Prerequisites" section that states what knowledge or skills the reader should have before starting the lesson.

### Hands-On Examples
- Include practical, executable code examples
- Provide clear explanations of what the code does
- Include exercises for readers to try
- Specify expected outcomes

### Exercises
- Include difficulty indicators (Beginner, Intermediate, Advanced)
- Provide clear instructions
- Include expected outcomes
- Suggest possible variations or extensions

## Writing Style

### Tone
- Maintain a beginner-friendly, encouraging tone
- Use clear, jargon-free language while maintaining precision
- Be supportive and patient with readers
- Include motivational elements

### Language
- Use active voice when possible
- Use clear, concise sentences
- Define technical terms when first used
- Avoid ambiguous language
